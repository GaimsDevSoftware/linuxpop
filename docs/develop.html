Build your own plugin

Two paths: the no-code wizard for common patterns, or a tiny Python file for full power. Both show up in the popup the same way.

TL;DR: if you want to "open this text in URL X" or "run this shell command on this text", use the wizard - 30 seconds, zero code. For anything else, the Python API is one decorator and a function.

Path 1 · Wizard (no Python)

The wizard handles the ~80% of plugin ideas that boil down to one of four primitives:

  • Open a URL with the selection inserted (Wikipedia search, Google Maps, YouTube, anything)
  • Run a shell command with the selection (custom scripts, pandoc, notify-send)
  • Show a notification with templated text
  • Copy a transformed string back to the clipboard

Open the wizard

  1. Click the tray icon → Plugins…
  2. Switch to the Custom tab
  3. Click New action…

Fields explained

Internal name - alphanumeric + hyphens. Becomes the filename and the plugin's ID.
Example: wikipedia-search

Button label - what the popup tooltip shows.
Example: Search Wikipedia

Icon - type a name from your icon theme, or click Browse… to pick from 3,800+ symbolic icons with live search. Want your own brand mark? Drop an .svg or .png in ~/.config/linuxpop/icons/ - it's searchable the next time you open the picker.

When clicked - pick one of the four action types.

Template - the actual payload. Use these variables:

{text}        the raw selection
{text_url}    URL-encoded (use this in href="...?q={text_url}")
{text_shell}  shell-quoted (use this inside a command line)
{text_upper}  uppercased
{text_lower}  lowercased
{text_strip}  whitespace-trimmed

Click any variable chip under the field to insert it at the cursor.

Show on - which content types should trigger this button. Leaving all unchecked means the button is always shown.

Example recipes

Wikipedia search

Name:      wikipedia-search
Label:     Search Wikipedia
Action:    Open a URL
Template:  https://en.wikipedia.org/wiki/Special:Search?search={text_url}
Show on:   Plain text

Translate to Norwegian via DeepL

Name:      deepl-no
Label:     Translate to Norwegian
Action:    Open a URL
Template:  https://www.deepl.com/translator#auto/no/{text_url}
Show on:   Plain text

Show text statistics

Name:      text-stats
Label:     Stats
Action:    Show a notification
Template:  Selection: {text_strip}  (length above)
Show on:   Plain text

Wrap text in markdown code-fence

Name:      md-code
Label:     Wrap as code block
Action:    Transform and copy
Template:  ```
{text}
```
Show on:   Plain text

Where it's saved

Each recipe is one JSON file in ~/.config/linuxpop/recipes/<name>.json. Share a recipe by sending the JSON - the recipient drops it in their recipes folder and it shows up next reload.

Path 2 · Python plugin

For anything the wizard can't express - HTTP calls, multi-step workflows, GUI dialogs, reading files - write a Python file in ~/.config/linuxpop/plugins/.

Minimal example

"""Plugin that shouts the selection in a notification."""
import subprocess

from classifier import ContentType
from plugin_base import Plugin


def _shout(text: str) -> None:
    subprocess.run(
        ["notify-send", "-i", "face-surprise", "SHOUT", text.upper()],
        check=False,
    )


def register(register_plugin) -> None:
    register_plugin(Plugin(
        name="shout",
        icon="face-surprise-symbolic",
        tooltip="SHOUT IT",
        handler=_shout,
        content_types=(ContentType.PLAIN_TEXT,),
        priority=50,
    ))

Save as ~/.config/linuxpop/plugins/shout.py, right-click tray → Plugins… → it's loaded. Select text → click the new button → notification.

The Plugin dataclass

Plugin(
    name=str,                  # unique id, used in logs and order config
    icon=str,                  # theme icon name (any "-symbolic" or full icon)
    tooltip=str,               # hover text on the popup button
    handler=Callable[[str]],   # called with the selected text on click
    content_types=Iterable[ContentType],   # () = always, or specific types
    priority=int,              # lower = earlier in popup (default 100)
)

ContentType values

  • ContentType.PLAIN_TEXT - anything that's not classified more specifically
  • ContentType.URL - looks like https://…, www.x.com, github.com/foo
  • ContentType.EMAIL - name@domain.tld
  • ContentType.PATH - starts with /, ~/, ./
  • ContentType.COMMAND - first token looks like a shell command (git, sudo, ls, …)

Tips

  • Run long work on a thread. Your handler is called on the GTK main loop, but LinuxPop wraps it in a worker thread for you already. Heavy HTTP calls are fine.
  • Copy results to the clipboard with subprocess.run(["xclip", "-selection", "clipboard"], input=text.encode()) so the user can paste them.
  • Use notify-send for any user feedback. LinuxPop's popup closes when the handler returns; notifications are the natural way to confirm.
  • Don't sys.path.insert - plugin_loader puts the LinuxPop directory on sys.path already.

Plugin settings

Your plugin can read user settings from settings.py's singleton:

from settings import get_settings
config = get_settings()
my_value = config.get("my_plugin_key", "default")

To make a setting visible in the LinuxPop Settings dialog, add a default to settings.py's DEFAULTS dict and a row in settings_gui.py. (Drop your file in the catalogue with a PR - we'll wire it up.)

Sharing your plugin

Two options:

  1. Recipe (.json): if it's a wizard recipe, just share the file. Anyone drops it in their ~/.config/linuxpop/recipes/.
  2. Python (.py): open a PR against plugins_repo/ in the LinuxPop repo with your file and an entry in manifest.json. We curate the catalogue.
Coming in v0.2: a community catalogue. PR your recipe or plugin to the LinuxPop repo and we'll feature it. Eventually a hosted Community tab in Plugin Manager will let users browse without git.
